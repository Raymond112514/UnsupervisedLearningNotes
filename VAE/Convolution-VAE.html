10. Convolutional VAE#

In a convolutional VAE, both the encoder and decoder are parameterized by convolutional and deconvolutional neural networks. The encoder \(q\) processes an image and produces a mean vector \(\mathbf{\mu}_{\phi}\) and a variance vector \(\mathbf{\sigma}_{\phi}\). The decoder \(p\) takes a latent vector as input and generates a mean vector \(\mathbf{\mu}_{\theta}\) in the image space, meaning it has the same dimensions as the original image. For the decoder, we fix the variance as an identity matrix.

\[\begin{split} \begin{align*} p_{\theta}(\mathbf{x}|\mathbf{z}) &= \mathcal{N}(\mathbf{x}|\mathbf{\mu}_{\theta}(\mathbf{z}), \mathbf{I})\\ q_{\phi}(\mathbf{z}|\mathbf{x}) &= \mathcal{N}(\mathbf{z}|\mathbf{\mu}_{\phi}(\mathbf{x}), \mathbf{\sigma}_{\phi}(\mathbf{x})^2 \mathbf{I})\\ p(\mathbf{z}) &= \mathcal{N}(\mathbf{z}|\mathbf{0}, \mathbf{I})\\ \end{align*} \end{split}\]

10.1. Convolutional VAE Architecture#

We implement and test a convolutional VAE on both the MNIST and CIFAR-10 datasets, following the architecture shown in Figure 1. During preprocessing, we normalize the images to the range \([-1, 1]\).

Figure 1 : Convolutional VAE encoder and decoder architecture.

For the loss function, we use mean square error loss directly in place of the original reconstruction loss. This is because \(p_{\theta}(\mathbf{x}|\mathbf{z}) = \mathcal{N}(\mathbf{x}|\mathbf{\mu}_{\theta}(\mathbf{z}), \mathbf{I})\), and therefore

\[\log p_{\theta}(\mathbf{x}|\mathbf{z})\propto -||x-\mathbf{\mu}_{\theta}(\mathbf{z})||_2^2\]

The prior matching loss remains the same, and we use the formula for KL-divergence between two multivariate Gaussian directly:

\[\mathcal{D}_{KL}(\mathcal{N}(\mu_1, \Sigma_1)||\mathcal{N}(\mu_2, \Sigma_2)) = \frac{1}{2}\bigg[\log \frac{|\Sigma_2|}{|\Sigma_1|}-d+(\mu_1-\mu_2)^T\Sigma_2^{-1}(\mu_1-\mu_2)+\text{Tr}(\Sigma_2^{-1}\Sigma_1)\bigg]\]

See [1] for a complete derivation.

class Encoder(nn.Module):
    """
    Implementation for the encoder q(z|x)
    """
    def __init__(self, in_channels, latent_dim):
        """
        @param in_channels: int
            Number of channels of the input image
        @param latent_dim: int
            Latent variable dimension
        """
        super(Encoder, self).__init__()
        self.latent_dim = latent_dim
        self.net = nn.Sequential(
            nn.Conv2d(in_channels, 32, 3, 1, 1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.Conv2d(32, 64, 3, 2, 1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.Conv2d(64, 128, 3, 2, 1),
            nn.BatchNorm2d(128),
            nn.ReLU(),
            nn.Conv2d(128, 256, 3, 2, 1),
            nn.BatchNorm2d(256),
            nn.ReLU(),
            nn.Flatten(),
            nn.Linear(4096, 2*latent_dim)
        )

    def forward(self, x):
        x = self.net(x)
        mu, log_std = torch.chunk(x, 2, dim=1)
        return mu, log_std

class Decoder(nn.Module):
    """
    Implementation for the decoder p(x|z)
    """
    def __init__(self, out_channels, latent_dim):
        """
        @param out_channels: int
            Number of channels of the reconstructed image
        @param latent_dim: int
        """
        super(Decoder, self).__init__()
        self.latent_dim = latent_dim
        self.net = nn.Sequential(
            nn.Linear(latent_dim, 2048),
            nn.ReLU(),
            nn.Unflatten(1, (128, 4, 4)),
            nn.ConvTranspose2d(128, 128, 4, 2, 1),
            nn.BatchNorm2d(128),
            nn.ReLU(),
            nn.ConvTranspose2d(128, 64, 4, 2, 1),
            nn.BatchNorm2d(64),
            nn.ReLU(),
            nn.ConvTranspose2d(64, 32, 4, 2, 1),
            nn.BatchNorm2d(32),
            nn.ReLU(),
            nn.Conv2d(32, out_channels, 3, 1, 1),
            nn.Tanh()
        )

    def forward(self, z):
        x = self.net(z)
        return x

class ConvolutionVAE(nn.Module):
    """
    Implementation of convolutional VAE class
    """
    def __init__(self, channels, latent_dim, device="cpu"):
        """
        @param channels: int
        @param latent_dim: int
        @param device: string
        """
        super(VAE, self).__init__()
        self.encoder = Encoder(channels, latent_dim)
        self.decoder = Decoder(channels, latent_dim)
        self.latent_dim = latent_dim
        self.device = device

    def forward(self, x):
        """
        @param x: torch.tensor [B, C, H, W]
        """
        self.channels, self.height, self.width = x.shape[1:]
        mu_x, log_std_x = self.encoder(x)
        z = mu_x + log_std_x.exp() * torch.randn_like(mu_x)
        x_recon = self.decoder(z)
        x_recon = torch.clamp(x_recon, -1, 1)
        return x_recon, mu_x, log_std_x

    def sample(self, n_samples):
        """
        @param n_samples: int
        """
        self.eval()
        z = torch.randn(n_samples, self.latent_dim).to(self.device)
        samples = self.decoder(z)
        samples = torch.clamp((samples + 1)/2, 0, 1)
        samples = samples.permute(0, 2, 3, 1)
        samples = samples.detach().cpu().numpy()
        return samples

def loss_fn(x, x_recon, mu, log_std):
    """
    Loss function for the vae
    @param x, x_recon: torch.tensor [B, C, H, W]
    @param mu: torch.tensor [B, L]
    @param log_std: torch.tensor [B, L]
    """
    recon_loss = F.mse_loss(x, x_recon, reduction='none').view(x.shape[0], -1).sum(dim=1).mean()
    kl_loss = -0.5 * torch.sum(1 + 2 * log_std - mu.pow(2) - (2 * log_std).exp(), dim=1).mean()
    loss = recon_loss + kl_loss
    return loss, kl_loss, recon_loss

10.4. Convolution VAE on CIFAR10#

We now train a VAE on the CIFAR-10 dataset, following the same architecture as before. Note that the generated samples appear blurry, a phenomenon observed in [2] and [3]. One reason for this is the use of the \(L_2\) norm in the reconstruction loss. The \(L_2\) norm is minimized when the possible outputs are averaged, leading to blurry results.

vae = VAE(256, device=DEVICE).to(DEVICE)
optimizer = optim.Adam(vae.parameters(), lr=1e-3)
train(vae, optimizer, train_loader, 10)
