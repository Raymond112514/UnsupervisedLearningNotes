2. CharRNN#

Recall that we want to model the conditionals

\[p(x_1, x_2,..., x_D) = \prod_{i} p(x_i|\mathbf{x}_{<i})\]

Jointly. Note that such models must be able to accept variable length inputs. One idea is to use a reccurent neural network to parameterize the conditionals. Conditioned on the previous variables \(x_1,..., x_{i-1}\), the RNN estimates the probability distribution of the next token.

Katex

Figure 1 : Architecture of the Character-level RNN.

One such instance of a RNN model is CharRNN, proposed by Andrej Karpathy in 2015. The model is originally trained on a corpus of text to generate new task. In this note, to save the computation, we simplify the task to generate binarized MNIST digits. To do so, we first preprocess the MNIST data so that each image is binarized to and one hot encoded.

import torch.nn as nn
import numpy as np
from torch.utils.data import Dataset, DataLoader, random_split
import torchvision.datasets as datasets
import torchvision.transforms as transforms
import torch.nn.functional as F
import matplotlib.pyplot as plt
from CharRNN import *

DEVICE = torch.device("cuda" if torch.cuda.is_available() else "cpu")

transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Lambda(lambda x: x.view(-1))
])

train_dataset = datasets.MNIST(root='./data', train=True, download=True, transform=transform)
test_dataset = datasets.MNIST(root='./data', train=False, download=True, transform=transform)

class MNISTDataset(Dataset):
    """
    Custom MNIST dataset: binarizes each image and one hot encodes the label
    """
    def __init__(self, data):
        self.data = data

    def __len__(self):
        return self.data.__len__()

    def __getitem__(self, idx):
        X, _ = self.data.__getitem__(idx)
        X = torch.where(X > 0, torch.tensor(1), torch.tensor(0))
        return F.one_hot(X[:-1], num_classes=2), X[1:]
    
dataset = MNISTDataset(train_dataset)
dataloader = DataLoader(dataset, batch_size=128, shuffle=True)

2.1. CharRNN Implementation#

For our implementation of the CharRNN, we follow a similar architecture as in the original CharRNN. We used 2-layer LSTM layers with 128 hidden dimensions and dropout \(0.2\). The output of the LSTM is then passed into a \(2\) layer feed forward layer with ReLU activation and 128 hidden units. The model outputs \(2\) real numbers, representing the logits for class \(0\) and \(1\).

class CharRNN(nn.Module):
    """
    Implementation of CharRNN for Binary MNIST data
    """
    def __init__(self, vocab_size, hidden_dim):
        """
        @param vocab_size (V): int
        @param embedding_dim (E): int
        @param hidden_dim (H): int
        @param length (L): int
        @param batch_size (B): int
        """
        super(CharRNN, self).__init__()
        self.rnn = nn.LSTM(vocab_size, hidden_dim, num_layers=2, batch_first=True, dropout=0.2)
        self.linear = torch.nn.Sequential(torch.nn.Linear(in_features=hidden_dim, out_features=hidden_dim),
                                           torch.nn.ReLU(),
                                           torch.nn.Linear(in_features=hidden_dim, out_features=vocab_size))

    def forward(self, x):
        """
        @param x: torch.tensor [B, L, 2]

        Shape after RNN: [B, L, H]
        Shape after out: [B, L, V]
        """
        x, _ = self.rnn(x)
        out = self.linear(x)
        return out

    def generate(self, n_samples, max_len=28**2, vocab_size=2, device="cpu"):
        """
        @param n_samples: int
            Number of samples to geneerate
        @param max_len: int
            Length of the generated sample
        @param vocab_size: int
        @param device: string
        """
        self.eval()
        samples = torch.zeros((n_samples, max_len), dtype=torch.long).to(DEVICE)
        for i in range(1, max_len):
            x = samples[:, :i]
            x = F.one_hot(x, num_classes=vocab_size).float().to(DEVICE)
            logits = self.forward(x)
            logits = logits[:, -1, :]
            prob = F.softmax(logits, dim=1)
            epsilon = torch.rand(n_samples, 1).to(DEVICE)
            characters = (epsilon >= prob[:, :1]).long()
            samples[:, i] = characters.squeeze(1)
        samples = samples.view(n_samples, int(max_len**0.5), int(max_len**0.5))
        samples = samples.detach().cpu().numpy()
        return samples

We train the model for around 200 epochs using Adam optimizer with learning rate of \(10^{-6}\). During the experiment, it is observed that learning rate that is too high (around \(10^{-3}\)) wouldn’t work. The model needs to be trained with much lower learning rate.

vocab_size = 2
hidden_dim = 2
charrnn = CharRNN(2, 128)
train(charrnn, dataloader, 20, device=DEVICE, lr=1e-6)

2.2. Sample generation#

After training, we see that overall the model is able to generate handwritten digits. However, occasionally the strokes become discontinued at the boundaries. This is expected since in CharRNN, we flatten the images, causing the boundary points of the MNIST images to be directly connected. Therefore, sometimes the model fails to notice the boundary of the image and continue the stroke after the boundary.

Figure 2 : Raster ordering. When flattened, the model treats pixel 5 directly connected to pixel 4, thereby causing discontinuities. 

n_samples = 36
samples = charrnn.generate(n_samples)
plot_samples(samples)

If we shift such image slightly, we see that we can recover the good digits.

2.3. CharRNN with positional encoding#

One way of resolving this issue is by introducing postional information so that the model knows when it encounter the boundaries. Instead of using the sinusoids to encode position as in transformers, we simply feed the \((x,y)\) coordinate into the model.

charrnn_with_pos = CharRNN(2, 128)
train_with_pos(charrnn_with_pos, dataloader, 20, device=DEVICE, lr=1e-6)

2.4. Sample generation with positional encoding#

After introducing the positional encoding, we see that the discontinuities disappears and the digits are centered to each frame.

num_samples = 36
samples = sample_with_position(charrnn_with_pos, num_samples, device=DEVICE)
plot_samples(samples)
